NYQP 2025 Animated Map Setup Instructions

Note: This setup creates an interactive animated map showing mobile station activity during the NYQP 2025 contest.

Prerequisites

  • Python 3.x - Built-in libraries only (sqlite3, json)
  • Contest log files - Cabrillo format (.log files)
  • Geographic data - NY state boundaries and county coordinates

Required Directory Structure

NYQP-2025/ ├── logs/ # Contest log files │ ├── ab1bl.log │ ├── k2a.log │ ├── k2g.log │ └── ... (other mobile logs) ├── analysis/ │ ├── scripts/ │ │ ├── new_generate_animated_map.py │ │ └── create_database.py # (to be created) │ └── data/ │ ├── ny_boundaries.geojson # (to be obtained) │ └── ny_counties.json # (to be created) └── nyqp_2025.db # SQLite database (generated)

Step-by-Step Setup

Step 1: Obtain Contest Log Files

Collect Cabrillo format log files from mobile stations that participated in NYQP 2025. The current script expects these mobile stations:

  • AB1BL, K2A, K2G, K2Q-R, K2V, KQ2R, KV2X-M
  • N1GBE, N2B, N2CU, N2T, W1WV-M, WI2M, WT2X

Place all .log files in the logs/ directory.

Step 2: Get NY State Geographic Data

NY State Boundaries (GeoJSON):

  • US Counties GeoJSON (filter for NY)
  • US Counties with FIPS codes
  • US Census Cartographic Boundary Files

Save as analysis/data/ny_boundaries.geojson

Step 3: Create County Coordinates File

Create analysis/data/ny_counties.json with county center coordinates:

{ "ALB": {"lat": 42.6803, "lng": -73.8370, "name": "Albany"}, "ALL": {"lat": 42.0931, "lng": -78.2003, "name": "Allegany"}, "BRX": {"lat": 40.8448, "lng": -73.8648, "name": "Bronx"}, ... }

Resources for county coordinates:

  • LatLong.net County Coordinates
  • SimpleMaps US Counties Database

Step 4: Create Database Creation Script

Create analysis/scripts/create_database.py to parse log files and create the SQLite database:

#!/usr/bin/env python3 import sqlite3 import os import re from datetime import datetime def parse_cabrillo_logs(): # Parse .log files and extract QSO data # Create tables: qsos (call, datetime, worked_county, log_file) # Insert QSO records with timestamps and county information pass if __name__ == "__main__": parse_cabrillo_logs()

Step 5: Modify the Animation Script

Update new_generate_animated_map.py to use direct file sources instead of the reference HTML:

  • Load boundaries from ny_boundaries.geojson
  • Load county coordinates from ny_counties.json
  • Generate mobile tracks directly from database QSO data
  • Remove dependency on reference HTML file

Step 6: Run the Setup

# 1. Create the database from log files cd NYQP-2025/analysis/scripts python create_database.py # 2. Generate the animated map python new_generate_animated_map.py

Output: nyqp_2025_mobile_activity.html
